  • Exploiter, dépanner et superviser une solution d’infrastructure réseau
Conditions de réalisation

Ressources fournies

  • Un environnement de virtualisation (Hyper-V)
  • Une machine virtuelle pfSense
  • Deux machines virtuelles sous Debian 13
  • Accès aux paquets logiciels (Apache, HAProxy, Suricata)

Résultats attendus

  • Mise en place d’une DMZ sécurisée
  • Déploiement de deux serveurs Web accessibles
  • Mise en place d’un système de répartition de charge
  • Assurer la haute disponibilité du service Web
  • Détection d’activités réseau suspectes (ex : ping)
Ressources documentaires, matérielles et logicielles

Logiciels utilisés

  • pfSense → routage, firewall, DMZ
  • HAProxy → load balancing
  • Suricata → détection d’intrusion
  • Apache → serveur Web
  • Debian 13 → système des serveurs
  • Hyper-V → virtualisation

Modalités d’accès aux productions

Le projet sera documenté dans un document Word.

Descriptif de la réalisation professionnelle

  • Installation de deux serveurs Web Debian avec Apache
  • Configuration du pare-feu pfSense avec plusieurs interfaces (LAN / WAN / DMZ)
  • Création d’une DMZ pour isoler les serveurs Web
  • Mise en place d’un Load Balancer avec HAProxy
  • Configuration de Suricata pour détecter les paquets ICMP (ping)
  • Tests de fonctionnement (accès web, répartition de charge, alertes IDS)
